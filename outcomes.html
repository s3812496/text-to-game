Project Motivation

Describe what motivated your group to choose this particular project?

The group collectively chose this project due to having a passion for video games. We grew up in an environment surrounded by video games and spent most of our childhood sharing laughter and joy with our family and friends through video games. Given the chance to come up with our very own project, we thought it would be a great idea to develop an idea which would provide other people who love games just like us to develop their own text-based game and share their creative imagination with the rest of the world. We felt that entry-level game-making was too hard to access for everyday people with no prior experience in programming or technology in general and as a result, our group wanted to be that bridge that would allow beginners the opportunity to cross the bridge into game development. The platform that we’re hoping to create would hopefully give birth to many new innovative games. The group’s collective love for games is what motivated us to choose this project and to put smiles on other people’s faces.

We asked around if anyone had any ideas for a new game and nothing came up until Jack William Brown mentioned a text- based interactive game which we all agreed was a good idea. None of us really wanted to come up with and write a story however so in the end we decided to create a tool that would make it easier for authors to make interactive stories. We were particularly interested in this project because we all loved the idea of making a game and a text-based game seemed the best fit. We thought that having to write a story ourselves would be too difficult for a team project. There would not be enough work to go around and writing a worthwhile story takes time. All these reasons motivated us to choose a project where the beginner programmers could learn while the more experienced members could be challenged as well as produce a service that could be used properly and potentially monetised.

How does it relate to your individual and collective interests? It is a laudable aim to try and create something which is beyond your current knowledge level and which addresses areas where you can enrich your IT Skills.

The programming in this project was divided specifically into stations. Nicholas Leppan(s3840050) and Nathan Martina(S3812496) took on the role of laying out what the program is going to require.

Although this is a programming heavy project, some group members such as myself (Anthony), have only a basic understanding of programming and as such my skills must be utilized elsewhere to fulfil my duty within the group.

We decided that the program would be developed in Java because all members have experience in the language. The project has been laid out to contain simple methods like displaying windows, text and buttons. With all the complex methods such as storing information and creating the mapping system used for all parent-child relations in chapters. The more complex programs will also be the ones that controls the flow of the program and the parameters used in each simple feature, this means that the more advanced programmers that will be working on the complex mappings will also need to supervise the information that will be go into and be returned by all other methods. This forces anyone working on the complex programs to help those working on smaller programs and means that every member will have to check in with progress often, so no one is left behind.

Furthermore, this would be a good learning experience for all group members involved as Nicholas and Nathan being the best programmers of the group can share their knowledge of programming to the rest of the members and vice-versa to create a well-rounded group who knows their strengths and weaknesses. Individually, this project serves to enrich other areas of IT in which we normally aren’t familiar with and helps each group member step outside their comfort zone and really delve into the deep end of the pool.

Overall, the main difficulty that will challenge the members is getting each feature to work together in one program. Also, this project is very much not suited to brute force solutions and requires more elegant ways to create the program. This is not easy for new programmers with little experience and will likely cause problems for all members later in the project.

Collectively, it would be in the group’s best interest to recruit members with many different strengths in order to evenly distribute the workload and allow those who are more sufficient at java work on the programming aspect of the project and those who specialise in design/admin take on that aspect of the project.

Minimum Viable Features

1 - Creating Chapters

Description of feature:

Users will be able to create new Chapters and will be able to enter text into the block which will be saved.

Description of test:

Check for no words, one paragraph, and an entire page of text. Record how the text prints in the window. Check against expected results and or errors.

2 - Creating options for next chapter

Description of feature:

Users will be able to add new child chapters to previously made chapters as one of the options for their interactive text.

Description of test:

Test can you move onto the next chapter without completing the current How many options can you create? Does each option keep its link to its parent chapter? Test error throwbacks and errors in the save system.

3 - Viewing created chapters in an easy to read tree map

Description of feature:

The user will be able to view what they have created in a laid-out map of the chapters and any chapters they may connect to. Users will be able to edit the connections and flow between chapters from this screen. User should be able to edit, move and delete as necessary

Description of test:

Can the map be edited? When changing the map will it edit the inner story Tested by simply moving and changing the connections between chapters
