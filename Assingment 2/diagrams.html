EVF-1 Saving File for reader

Description of EVF-1

  • When reading the stories created by users, the program should save where in the story the reader is and what choices have been made so far. 
  • We will test whether the chapter automatically saves into our database. This will also test whether the saves will automatically overwrite and save any previous progress. 

User Story

As a creator using Text-to-Game, I should be able to confirm that my program saves where the reader last was and what choices have been made so far when they are playing the game. The users playing my game should not have to start from the very beginning each time they want to play. When they play a game made by Text-to-Game, there should be a correct sense of time and progression.

Design Diagrams

Lucid Textual Explanation

When playing a chapter as a player, the autosave feature will periodically save the user’s progress. This feature helps to minimise data loss and attempts to recover game progress in case of a system failure or crash. The autosave feature creates a save state periodically/upon every change within the game and stores it into a database. When the user continues playing the game, the game retrieves the stored data and loads the most recent save state. It is important that a system reliably functions as intended and that information being stored and retrieved correctly.

Validation Testing

  • The program correctly stores data into the save file where the reader last was and their previous choices within the game
  • he program periodically overwrites to the save file
  • The program correctly retrieves information from the save file, loading the it and displaying the game
  • The program automatically saves when closed
